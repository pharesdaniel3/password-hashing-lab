# Module 1: The One-Way Function

A cryptographic hash is a one-way street. You can turn a password into a hash, but you can't turn the hash back into the password.

e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855
Observation: Notice how even a tiny change in the input (like adding a period) completely changes the SHA-256 hash. This is called the Avalanche Effect.

🧂 Module 2: Why We Need Salt

If two users have the same password, they have the same hash. A "Salt" is random data added before hashing to ensure unique results.

The Defense: With salts, even if 1,000 users use the same password, their stored hashes will all be different. This stops Rainbow Table attacks.

🌈 Module 4: Rainbow Table Attack

Attackers use massive pre-computed tables of hashes to "look up" common passwords instantly. Salts make these tables useless.
